# Feature Development Workflow

Use this sequence for normal feature work:

1. Read project context and product principles.
2. Read the current feature spec.
3. Confirm scope and non-goals.
4. Model domain rules before UI.
5. Add tests around pure logic first when behavior is non-trivial.
6. Implement UI with loading, empty, error, and success states.
7. Add i18n messages.
8. Run verification.
9. Update specs and docs if behavior changes.
