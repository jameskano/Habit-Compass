# Feature Development Workflow

Use this sequence for normal feature work:

1. Read project context and product principles.
2. Read `docs/engineering/ai-code-task-guardrails.md`.
3. Read the current feature spec.
4. Confirm scope and non-goals.
5. Model domain rules before UI.
6. Add tests around pure logic first when behavior is non-trivial.
7. Implement UI with loading, empty, error, and success states.
8. Add i18n messages.
9. Run verification.
10. Update specs and docs if behavior changes.
