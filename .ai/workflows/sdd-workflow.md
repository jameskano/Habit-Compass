# SDD Workflow

1. Identify the relevant MVP or feature spec.
2. Read `docs/engineering/ai-code-task-guardrails.md`.
3. If no spec exists, create one from `specs/_templates/feature-spec.md`.
4. Write requirements and acceptance criteria before implementation.
5. Define domain behavior separately from UI behavior.
6. Implement the smallest useful slice.
7. Add or update tests for domain behavior and user-facing flows.
8. Run review gates and verification commands.
9. Update documentation if architecture, behavior, or workflow changes.
