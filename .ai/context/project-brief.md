# Project Brief

## Summary

Habit Compass is a habit, task, and recurrent-task tracker inspired by simple habit apps while leaving room for deeper planning based on roles, values, weekly planning, mood, reflection, and identity-oriented change.

## MVP Goal

Ship a fast daily tracker where users can create items, complete them, review simple stats, and manage settings without needing to understand the deeper planning system.

## User Types

- Simple tracker user: wants quick habit and task completion.
- Planner user: wants weekly planning and category context.
- Reflective user: wants optional mood and reflection prompts.
- Growth-oriented user: wants identity, values, and future suggestion support.

## Main Navigation

- Today/Home
- Week/Planning
- Items
- Settings
- Onboarding for first-run setup

Mood remains future optional depth rather than a current primary navigation item.

## Feature Philosophy

The default experience should be lightweight and obvious. Deeper features should be discoverable, optional, and easy to ignore.

## Optional Feature Principle

Optional depth must never block core habit tracking. A user who never configures roles, values, mood, reflections, or suggestions should still have a complete product.

## Future Expansion Path

The product can later grow into routines, projects, systems, Google Calendar integration, AI suggestions, subscriptions, and native mobile builds. Each expansion requires its own spec and review gate.
